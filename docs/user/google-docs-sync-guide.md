## はじめに

Meeting Minutes Automatorは、Google Meetの会議中に文字起こしした内容をリアルタイムでGoogle Docsに保存できます。

### 主な機能

- **リアルタイム同期**: 文字起こし結果が2秒以内にGoogle Docsに反映
- **オフライン対応**: ネットワーク切断時も自動的にキューに保存、復帰後に同期
- **カスタマイズ可能**: タイムスタンプ表示、バッファリング時間などを調整可能

## Google連携の設定

### Step 1: Chrome拡張機能を開く

1. Chromeのツールバーで **Meeting Minutes Automator** アイコンをクリック
2. ポップアップが表示されます

### Step 2: Googleアカウントと連携

1. **「Google アカウントと連携」** ボタンをクリック
2. Googleアカウント選択画面が表示されます
3. 使用するGoogleアカウントを選択
4. 権限の確認画面で **「許可」** をクリック

### Step 3: 連携完了の確認

- ステータスバッジが **緑色** に変わります
- 「Google Docs: 連携済み」と表示されます
- 設定セクションが表示されます

### 連携を解除する場合

1. ポップアップを開く
2. **「連携を解除」** ボタンをクリック
3. 確認後、連携が解除されます

## ドキュメントIDの取得方法

Google Docsに同期するには、同期先ドキュメントのIDが必要です。

### 方法1: URLから取得

Google DocsのURLは以下の形式です：

```
https://docs.google.com/document/d/【ここがドキュメントID】/edit
```

**例:**
```
https://docs.google.com/document/d/1aBcDeFgHiJkLmNoPqRsTuVwXyZ/edit
                                   ^^^^^^^^^^^^^^^^^^^^^^^^
                                   この部分がドキュメントID
```

### 方法2: 新しいドキュメントを作成

1. [Google Docs](https://docs.google.com) を開く
2. **「空白」** をクリックして新しいドキュメントを作成
3. ブラウザのアドレスバーからURLをコピー
4. 上記の方法でドキュメントIDを抽出

## 同期設定のカスタマイズ

Google連携後、以下の設定をカスタマイズできます。

### Google Docs同期

- **ON** (デフォルト): 文字起こしをGoogle Docsに自動保存
- **OFF**: 同期を一時停止

### タイムスタンプ表示

- **ON** (デフォルト): 各行に時刻を表示（例: `[14:30:25]`）
- **OFF**: 時刻を非表示

### 話者名表示

- **ON**: 話者名を表示（実験的機能）
- **OFF** (デフォルト): 話者名を非表示

### バッファリング時間

同期前の待機時間を1〜5秒で調整できます。

- **短い (1-2秒)**: より即時的な同期、API呼び出しが増加
- **標準 (3秒)**: バランスの取れた設定（デフォルト）
- **長い (4-5秒)**: API呼び出しを削減、まとめて同期

## トラブルシューティング

### 認証に失敗する

**症状**: 「認証に失敗しました」エラーが表示される

**解決方法**:
1. ブラウザのキャッシュをクリア
2. Chrome拡張機能を再読み込み（`chrome://extensions` → 更新ボタン）
3. 再度「Google アカウントと連携」をクリック

### 同期されない

**症状**: 文字起こし結果がGoogle Docsに反映されない

**確認事項**:
1. ステータスバッジが緑色（連携済み）か確認
2. 「Google Docs同期」設定がONか確認
3. ネットワーク接続を確認
4. DevTools Console (F12) でエラーを確認

### オフライン時の動作

**症状**: ネットワーク切断中に同期が止まった

**解決方法**:
- オフライン中の文字起こしは自動的にキューに保存されます
- ネットワーク復帰後、自動的に再同期されます
- 手動での操作は不要です

### 「ドキュメントが見つかりません」エラー

**原因**:
- ドキュメントIDが間違っている
- ドキュメントへのアクセス権がない

**解決方法**:
1. ドキュメントIDを再確認
2. Google Docsでドキュメントを開けるか確認
3. 共有設定で「編集者」権限があるか確認

## FAQ

### Q: 複数のドキュメントに同期できますか？

A: 現在のバージョンでは、一度に1つのドキュメントへの同期のみサポートしています。ドキュメントを切り替える場合は、新しいドキュメントIDを入力してください。

### Q: 同期されたデータはどこに保存されますか？

A: 文字起こしデータは指定したGoogle Docsドキュメントに直接保存されます。ローカルには一時的なキューのみが保存され、同期完了後に削除されます。

### Q: オフライン時のキューはどのくらい保持されますか？

A: Chrome拡張機能のローカルストレージ（最大5MB）に保存されます。通常の使用では数時間分の文字起こしを保持できます。

### Q: 同期の頻度はどのくらいですか？

A: バッファリング時間（デフォルト3秒）ごとに同期されます。Google Docs APIのレート制限（60リクエスト/分）を超えないよう自動調整されます。

### Q: 連携を解除するとデータは消えますか？

A: 連携を解除しても、すでにGoogle Docsに保存されたデータは残ります。未同期のキューにあるデータはローカルに保持されますが、再連携するまで同期されません。

### Q: 会議中に連携を切り替えても大丈夫ですか？

A: はい、会議中でも連携の切り替えは可能です。ただし、同期がOFFの間の文字起こしはキューに保存されません。
